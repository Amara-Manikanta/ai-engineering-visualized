RAG Type

Corrective RAG (CRAG)

Self-correcting retrieval that falls back to web searches.

How it Works

CRAG introduces an evaluator model right after retrieval. It grades the retrieved documents:

  • Correct: Proceed to generation.
  • Ambiguous: Extract what is useful, discard the rest.
  • Incorrect: Discard the retrieval and automatically trigger a live Web Search (e.g. Tavily/Google) to find the right answer.

Example Use Case

Customer support bots where internal docs might be outdated. If it can't find a confident answer internally, it searches the live public website before answering.

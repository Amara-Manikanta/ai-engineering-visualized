Part 4

Evaluating a RAG System

How do you know if your RAG pipeline is actually working? You need robust, automated evaluation metrics to measure retrieval quality and generation faithfulness.

28. Core Evaluation Metrics

🎯

Context Relevance (Precision)

Did the retriever find the right chunks? Or did it pull in noisy, irrelevant data? High precision means every retrieved chunk is useful.

🧲

Context Recall

Did the retriever find ALL the necessary information to answer the question? High recall means nothing important was missed.

🛡️

Faithfulness (Groundedness)

Is the final answer derived STRICTLY from the retrieved context? If the LLM makes up a fact not found in the chunks, faithfulness fails (Hallucination).

✅

Answer Relevance

Does the final answer actually address the user's original query? The answer might be faithful to the context, but completely useless to the user.

29. Common Problems in RAG

⚠️ The "Lost in the Middle" Phenomenon

When you feed an LLM a massive context window filled with dozens of retrieved chunks, it tends to pay attention to the chunks at the very beginning and the very end, while ignoring the chunks in the middle.

  • Poor Retrieval: The vector database fails to find the right chunk because the query didn't semantically match the document text (solved by query expansion or Hybrid RAG).
  • Bad Chunking: Chunks are cut mid-sentence or mid-paragraph, stripping away vital context (solved by semantic or structural chunking).
  • Duplicate Chunks: The same information is retrieved multiple times from slightly different pages, wasting context window space.
  • Hallucinations: The LLM ignores the prompt instructions and relies on its own pre-trained memory to answer the question.

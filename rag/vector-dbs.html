Vector Databases

Traditional databases search by keywords. Vector databases search by meaning.

13. What is a Vector Database?

A vector database is a specialized database designed to store, manage, and search embedding vectors. Instead of matching exact strings like SQL, a vector database uses Approximate Nearest Neighbor (ANN) algorithms to find vectors that are mathematically closest to your query vector.

💡 Why use one?

At scale, computing cosine similarity between your query and a million document chunks is too slow. Vector databases use specialized indexes (like HNSW) to reduce search times from minutes to milliseconds.

14. Popular Vector Databases

Pinecone

Fully managed, closed-source SaaS. extremely popular for its ease of use and serverless architecture.

Milvus

Open-source, highly scalable. Built for massive enterprise workloads handling billions of vectors.

Chroma

AI-native, open-source database optimized for developer experience. Great for rapid prototyping.

Qdrant

Written in Rust, extremely fast, open-source, and supports advanced payload filtering.

FAISS

A library by Meta. Not a full database, but the underlying engine for rapid vector similarity search.

pgvector

An extension for PostgreSQL. Allows you to keep your vectors right next to your relational data.

15. Indexing Methods

How do vector databases achieve millisecond search times over billions of vectors? By using specialized indexes.

📏

Flat (Exact Search)

Compares the query to every single vector (brute force).

Pros: 100% accurate (perfect recall).
Cons: Extremely slow at scale.

🕸️

HNSW (Hierarchical Navigable Small World)

Builds a multi-layered graph of vectors. Search drops down through layers to quickly zoom in on neighbors.

Pros: Very fast, highly accurate.
Cons: High memory consumption.

🗂️

IVF (Inverted File Index)

Clusters vectors into groups (Voronoi cells). Search only checks the closest cluster.

Pros: Speeds up search significantly.
Cons: Can miss edge-case vectors.

📉

PQ (Product Quantization)

Compresses vectors by splitting them into sub-vectors and replacing them with short codes.

Pros: Massive memory reduction (up to 97%).
Cons: Lossy compression degrades accuracy.

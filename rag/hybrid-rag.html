RAG Type

Hybrid RAG

Combines semantic vector search with traditional keyword search.

How it Works

Dense vector embeddings are great for understanding meaning, but terrible at finding exact matches (like serial numbers or specific names). Hybrid RAG performs two searches simultaneously:

  • Dense Search (Vectors): Finds conceptually similar text.
  • Sparse Search (BM25/TF-IDF): Finds exact keyword matches.

The results are then merged using techniques like Reciprocal Rank Fusion (RRF).

Example Use Case

A medical research assistant. If a doctor searches for "Aspirin dosage for patient XYZ-992", BM25 catches "XYZ-992" exactly, while Vectors catch "dosage" semantics.

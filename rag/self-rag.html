RAG Type

Self-RAG

An LLM trained to critique its own retrieval and generation.

How it Works

Self-RAG involves fine-tuning an LLM to output special "Reflection Tokens". It actively decides:

  1. Do I need to retrieve information for this?
  2. Is the retrieved context relevant?
  3. Is my generated answer fully supported by the context (no hallucinations)?
  4. Is my generated answer actually useful to the user?

If it fails any of these self-checks, it loops back and tries again.

Example Use Case

Medical or Legal AI assistants where hallucinations are catastrophic. The model forces itself to prove that every sentence it generates is directly backed by the retrieved case law.

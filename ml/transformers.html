What is Generative AI?

Generative AI involves models that can create new data that resembles a training dataset — from text, to images, music, and code.

Generative AI models are trained to understand patterns in data and produce outputs that look and feel like the real thing. Unlike traditional AI that classifies or predicts, Gen AI creates.

The key breakthrough was the Transformer architecture (2017), which allowed models to process entire sequences in parallel and understand long-range dependencies — making large-scale generation possible.

Different Types of AI

AI is not a monolith. Different AI systems are designed for fundamentally different purposes.

🔬

Diagnostic / Descriptive

Analyzes historical data to understand what happened and why. Focuses on assessing correctness of past behavior.

EX: Fraud detection reports
📈

Predictive AI

Forecasts future outcomes based on historical patterns. Concerned with answering: What will happen next?

EX: Stock price models
🎯

Prescriptive AI

Determines the optimal course of action by providing data-driven recommendations.

EX: Route optimization
✨

Generative / Cognitive

Produces new content: code, articles, images, music. This is what powers ChatGPT, DALL·E, and Suno.

EX: GPT-4, Stable Diffusion
⚡

Reactive AI

Responds to specific inputs with predetermined responses. No memory, no learning from experience.

EX: Deep Blue chess engine
🧠

Limited Memory AI

Uses past experiences to inform current decisions. Modern ML models fall into this category.

EX: Self-driving cars
💡

Narrow AI (Weak AI)

Designed to perform a specific, limited range of tasks. Highly capable within its domain but nothing else.

EX: Image classifiers
🌐

General AI (Strong AI)

Can understand, learn, and apply knowledge across a wide range of tasks like human intelligence. Still theoretical.

EX: Hypothetical AGI

What are Language Models?

Generative AI applications are powered by language models — specialized ML models for natural language processing.

📖

A Language Model (LM) is a machine learning model trained to understand, generate, and manipulate natural language. At its core, it predicts the next token in a sequence given all the previous ones.

Step 1 · Document Ingestion

The model is trained with a large volume of natural language text — from the internet, books, and code. This corpus teaches the model patterns in language.

Step 2 · Encoder

The encoder takes input token sequences and uses the self-attention mechanism to determine relationships between tokens — which words influence which other words in the same sequence.

Step 3 · Embeddings

The encoder's output is a collection of vectors (multi-valued numeric arrays) where each element represents a semantic attribute of a token. Related words cluster near each other in this high-dimensional space.

Step 4 · Decoder

The decoder works on a new sequence of tokens and uses the embeddings generated by the encoder to produce an appropriate natural language output. It uses cross-attention to reference the encoder's context.

Step 5 · Generation

Given "When my dog was", the model uses attention over the input tokens and the encoder embeddings to predict the most probable next token — and continues until it generates a complete sequence like "a puppy".

📝

Splits on whitespace. Simple but fails on punctuation and new words.

Embeddings

Embeddings represent complex data as numerical vectors — capturing meaning so machines can process it mathematically.

An embedding is a dense vector of floating-point numbers. The key property is that semantically similar items produce similar vectors.

Word2Vec famously discovered that:

king − man + woman ≈ queen

This arithmetic works because the embedding space encodes meaning — not just individual words, but the relationships between them.

Word Embedding with Word2Vec 
from gensim.models import Word2Vec

# Sample sentences
sentences = [["I", "love", "NLP"],
             ["Transformers", "are", "amazing"]]

# Train Word2Vec model
model = Word2Vec(sentences, vector_size=50,
                 window=5, min_count=1)

# Get the embedding for a word
embedding = model.wv["NLP"]
print(embedding)  # [0.23, -0.11, ...]

The Attention Mechanism

How Attention Works

  1. Q
    Query — the current token asking "who should I pay attention to?"
  2. K
    Key — each token broadcasts what kind of information it holds.
  3. V
    Value — the actual information to retrieve if attention is high.
  4. ⊗
    Score — Q·KT / √d gives us how much each token should contribute.
Attention Formula
Attention(Q,K,V) = softmax( QKT / √dk ) · V

Multi-Head Attention

Instead of running attention once, the Transformer runs it multiple times in parallel, each with different learned weight matrices. This lets the model attend to different types of relationships simultaneously.

Head 1
Syntactic roles
Head 2
Coreference
Head 3
Positional proximity
Head 4
Semantic similarity

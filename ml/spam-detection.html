NLP Application

Spam Detection

Building a classifier to identify spam messages (Ham vs Spam) using Naive Bayes and NLP preprocessing.

Pipeline Overview

A typical Spam Detection model follows these core steps:

1. Preprocessing

Cleaning the dataset (e.g., SMSSpamCollection). Removing punctuation, lowercasing text, and tokenizing sentences.

2. Train/Test Split

Dividing the dataset to evaluate the model on unseen data.

3. Feature Maps (BoW / TF-IDF)

Converting the cleaned text into numerical vectors so the machine learning model can process it.

4. Model Training & Evaluation

Training a Naive Bayes Classifier and evaluating metrics like Precision, Recall, and Accuracy.

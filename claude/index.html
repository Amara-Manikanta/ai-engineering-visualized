Claude Code

12 Features Every Engineer Must Know

Master your AI coding assistant with these powerful workflows and configuration options.

1 CLAUDE.md
📄

Project memory file — custom rules, stack info, and commands Claude reads every session.

2 Permissions
🔒

Control what Claude can and can't touch — whitelist or block tools per session.

3 Plan Mode
📝

Review the plan before Claude acts — approve, edit, or reject each step first.

4 Checkpoints
⏱️

Revert to any point in time — automatic git snapshots let you undo any mistake.

5 Skills
🛠️

Reusable instructions Claude follows automatically — store in .claude/skills/.

6 Hooks
⚡

Run custom logic at key moments — shell scripts on PreToolUse, PostToolUse events.

7 MCP
🔌

Connect Claude to any external tool — databases, APIs, and services via Model Context Protocol.

8 Plugins
🧩

Extend Claude with third-party integrations — add tools without writing custom code.

9 Context
🧠

Feed Claude what it needs and manage the current context — files, history, and rules.

10 Slash Commands
⌨️

Trigger actions with a single command — store templates in .claude/commands/.

11 Compaction
🗜️

Compress long conversations to save tokens — keep context fresh without losing key info.

12 Subagents
🤖

Spawn parallel agents for complex tasks — divide and conquer multi-step workflows.
